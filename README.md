# คู่มือการติดตั้งและใช้งานระบบ SUB HDC (สำหรับลูกค้า)

ยินดีต้อนรับสู่ระบบ **SUB HDC** — ระบบนำเข้าและตรวจสอบข้อมูลสุขภาพระดับท้องถิ่นสำหรับไฟล์ F43 ZIP

ระบบนี้ได้รับการออกแบบให้อยู่ในรูปแบบของ **Docker Containers** เพื่อให้ง่ายต่อการติดตั้งและใช้งาน โดยประกอบไปด้วยสองส่วนหลักคือ:
1. **Next.js Web Application**: โปรแกรมเว็บแอปพลิเคชันสำหรับแสดงผลหน้าจออัปโหลดและหน้าแสดงประวัติผลการนำเข้าข้อมูล (ทำงานบนพอร์ต **80** ของเครื่องโฮสต์)
2. **MariaDB Database**: ระบบฐานข้อมูลสำหรับจัดเก็บข้อมูลชุดนำเข้า F43 (ทำงานบนพอร์ต **3308** ของเครื่องโฮสต์)

---

## 📋 สิ่งที่ต้องติดตั้งล่วงหน้า (Prerequisites)

ก่อนเริ่มติดตั้งระบบ กรุณาตรวจสอบให้แน่ใจว่าเครื่องของคุณได้ติดตั้งโปรแกรมเหล่านี้แล้ว:
1. **Docker Desktop** (พร้อมเปิดใช้งาน Docker Compose) -> [ดาวน์โหลดที่นี่](https://www.docker.com/products/docker-desktop/)
2. **Git** (สำหรับดึงโค้ดเวอร์ชันล่าสุด) -> [ดาวน์โหลดที่นี่](https://git-scm.com/)

---

## 🚀 ขั้นตอนการติดตั้งและรันระบบ

### ขั้นตอนที่ 1: ดึงโค้ดโครงการจาก Repository
เปิด Terminal หรือ PowerShell ของคุณในโฟลเดอร์ที่ต้องการติดตั้งระบบ แล้วรันคำสั่ง:

```bash
git clone https://github.com/tehnplk/SUB_HDC.git
cd SUB_HDC
```

### ขั้นตอนที่ 2: ตั้งค่าไฟล์คอนฟิกสภาพแวดล้อม (.env)
คัดลอกไฟล์ต้นแบบคอนฟิกไปสร้างเป็นไฟล์ `.env` ที่พร้อมใช้งานภายในโฟลเดอร์ `webapp/`:

```bash
cp webapp/.env.example webapp/.env
```
*(สำหรับ Windows PowerShell ให้ใช้คำสั่ง `copy webapp\.env.example webapp\.env`)*

เปิดไฟล์ `webapp/.env` ด้วยโปรแกรมแก้ไขข้อความ (Text Editor) เพื่อตรวจสอบหรือแก้ไขคอนฟิก เช่น:
* `ENCRYPT_KEY`: คีย์สำหรับเข้ารหัสข้อมูลลับทางการแพทย์ (AES-256-GCM)
* `JWT_KEY`: คีย์สำหรับลงนามเข้าใช้งาน API อย่างปลอดภัย

> [!WARNING]  
> เพื่อความปลอดภัยในการนำไปใช้งานจริง กรุณาเปลี่ยนค่า `ENCRYPT_KEY` และ `JWT_KEY` ให้เป็นรหัสส่วนตัวเฉพาะของคุณ

### ขั้นตอนที่ 3: สั่งรันระบบผ่าน Docker Compose
ที่โฟลเดอร์หลักของโครงการ ให้รันคำสั่งเพื่อสร้างและเปิดใช้งานระบบ:

```bash
docker compose up -d
```

**คำสั่งนี้จะทำสิ่งต่อไปนี้อัตโนมัติ:**
1. ดาวน์โหลดและเปิดใช้งานฐานข้อมูล MariaDB พร้อมสร้างโฟลเดอร์สำหรับเก็บข้อมูลถาวรที่โฟลเดอร์ `./mysql_data` บนเครื่องของคุณ
2. รันสคริปต์เริ่มต้นโครงสร้างฐานข้อมูลตาราง F43 จากไฟล์ `init_db/sub_hdc.sql` ในการรันครั้งแรก
3. ดาวน์โหลดแพ็กเกจและทำการคอมไพล์ระบบ Web App เวอร์ชันพร้อมใช้งานจริง (Production Build)

## 🔍 การตรวจสอบผลการรันระบบ

เมื่อรันเสร็จสิ้น คุณสามารถตรวจสอบสถานะการทำงานได้ด้วยคำสั่ง:

```bash
docker compose ps
```

* **หน้าเพจระบบ (Web App UI)**: เข้าใช้งานผ่านบราวเซอร์ที่ลิงก์ [http://localhost](http://localhost) (พอร์ต 80)
* **ฐานข้อมูล (Database Access)**: สามารถเชื่อมต่อฐานข้อมูลได้โดยตรงที่พอร์ต **3308** (`localhost:3308`, User: `root`, Password: `รหัสที่คุณระบุในไฟล์ .env`)

## 🔄 วิธีอัปเดตระบบเมื่อผู้พัฒนามีการแก้ไขโค้ด

หากทีมพัฒนาทำการอัปเดตตัวระบบขึ้น GitHub และคุณต้องการดึงเวอร์ชันล่าสุดมาใช้งาน ให้รันคำสั่งเหล่านี้:

```bash
# 1. ดึงโค้ดล่าสุดจาก GitHub
git pull

# 2. Rebuild และ Restart ตัว Web App
docker compose up -d --build webapp
```

> [!NOTE]  
> การอัปเดตด้วยวิธีนี้จะรวดเร็วและ**ปลอดภัยต่อข้อมูลเก่าในฐานข้อมูลอย่างสมบูรณ์** (ข้อมูลการนำเข้าเดิมจะไม่สูญหาย เนื่องจากระบบจะไม่ปิดตัวและไม่เขียนทับฐานข้อมูลเดิม)

## 📁 โครงสร้างโฟลเดอร์หลักของระบบ
* `/webapp` - ซอร์สโค้ดหลักของโปรแกรม Next.js ทั้งหมด
* `/webapp/tmp` - โฟลเดอร์ที่เก็บไฟล์ ZIP ที่อัปโหลดชั่วคราวและไฟล์ล็อกความผิดพลาด (ไม่บันทึกขึ้น Git)
* `/mysql_data` - โฟลเดอร์ที่จัดเก็บไฟล์ข้อมูลจริงของฐานข้อมูล MariaDB ในเครื่องของคุณ (ไม่บันทึกขึ้น Git)
* `/init_db` - เก็บไฟล์สคริปต์ SQL เริ่มต้นฐานข้อมูล
